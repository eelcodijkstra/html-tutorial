JavaScript voor interactie

In dit tutorial maken we kennis met Javascript, om interactie toe te voegen aan een webpagina.

In dit eenvoudige voorbeeld plaatsen we de Javascript-code direct in het HTML-bestand. Later zullen we zien dat het vaak handiger is om deze code in een apart bestand te plaatsen.

We kunnen JavaScript gebruiken voor de interactie met de gebruiker. We koppelen bijvoorbeeld een actie aan een knop. In dit tutorial gevel we meer voorbeelden.

We kunnen Javascript ook gebruiken voor de interactie met de eigen server, of met een andere webserver. Dit behandelen we in de module JS-0.

In die module leggen we veel meer uit over JavaScript. In dit tutorial is het voldoende als je kleine experimenten met de code kunt doen. Je hoeft niet alles helemaal te begrijpen.

DOM: HTML-elementen in Javascript

Om de interactie tussen JavaScript en het HTML-document in de browser mogelijk te maken, moet dit document zichtbaar zijn in de JavaScript-omgeving. Dit gebeurt in de vorm van de DOM: het document object model. Om een element hierin te vinden, kunnen we onder andere de id van dit element gebruiken (net als in CSS).

Het element met id="myElt" vinden we via: 

        document.getElementById("myElt")

Het resultaat hiervan is een JavaScript-object. Een HTML-attribuut met naam attr zien we dan als JavaScript-property met diezelfde naam, bijvoorbeeld: elt.attr.

We kunnen de waarde van een attribuut via dit element opvragen of veranderen. Dit laatste kan bijvoorbeeld via de toekenning: 

document.getElementById("myElt").style = "";

We kunnen zo ook de CSS-properties aanpassen, bijvoorbeeld style.color of style.backgroundColor.

Definitie van een functie

Een functie (function) is een beschrijving van een reeks acties of berekeningen. De definitie van een functie in JavaScript heeft de vorm: 

        function fname (p1, p2, p3) {
          ... acties of berekening met p1, p2 en p3 ...
        }

De naam van de functie, hier fname, gebruiken we om in de rest van programma naar deze functie te kunnen verwijzen.

De parameters van de functie, hier p1, p2, p3, gebruiken we in de acties of berekeningen. Deze bepalen mede het uiteindelijke resultaat.

De lijst met parameters kan leeg zijn, in dat geval schrijven we alleen de haakjes ().

Het gedeelte tussen accolades {...} noemen we de body van de functie.

Een functie is alleen een beschrijving van acties of berekeningen: deze acties of berekeningen worden nog niet uitgevoerd. Je kunt het vergelijken met een recept: pas als de kok eraan begint, gebeurt er iets.

Opdrachten

  1. Bekijk de definitie van de functie alert1. Hoeveel parameters heeft deze functie? Wat betekent de actie in de body? (Hint: zoek op het web naar "javascript alert".)
  2. Definieer naar analogie van de functie alert1 een tweede functie, alert2. Pas de tekst in de body aan, en zorg ervoor dat deze duidelijk verschilt (niet alleen in een enkel cijfer).

    Let op! Als je een fout maakt in de notatie, zal Thimble je daar niet op wijzen (anders dan in de HTML-code). Je krijgt geen foutmelding: het werkt alleen niet, of anders dan je denkt. Later zullen we hulpmiddelen gebruiken die meer hulp bieden.

We willen een actie koppelen aan een knop (button-element). We definiëren deze button in de body, voor het script-element. We geven deze button een naam (id), zodat we in de Javascript hiernaar kunnen verwijzen. (Vgl. het gebruik van id's voor CSS.)

Commentaar in JavaScript

Ook voor JavaScript geldt: je schrijft de code niet alleen voor de browser om uit te voeren. Ook mensen moeten je code kunnen lezen en begrijpen. Het is daarom belangrijk om je code netjes en overzichtelijk te houden, bijvoorbeeld door het gebruik van witruimte (inspringen, regelovergangen).

Soms moet je het hoe en waarom van de code uitleggen. Hiervoor maak je gebruik van commentaar. In Javascript heb je twee vormen van commentaar: (i) na de tekens // tot het eind van de regel; (ii) tussen de tekens /* en */. Dit gebruik je voor commentaar dat over meerdere regels loopt. 

        var pi = 3.1415926; // een benadering van pi
             ...
        /*
          Dit is een langer commentaar.
          Probeer het ook niet te lang te maken.
        */

N.B. De commentaar-regels voor HTML, CSS en Javascript zijn verschillend. Haal ze niet door elkaar!

  2. Wat gebeurt er als je HTML-commentaar binnen het Javascript-deel gebruikt, of Javascript-commentaar in het HTML-deel?
